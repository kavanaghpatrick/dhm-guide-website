## PATTERN 2: PERSONALIZATION/SEGMENTATION TABLES

### How It Works

High-performing content doesn't give ONE answer. It gives MULTIPLE ANSWERS segmented by:
- User weight/size
- Use case/drinking intensity
- Budget/income level
- Population type (age, gender, condition)

### Dosage Guide Example (Best Practice)

**Line 32-44 in dosage guide JSON:**

```markdown
| **Your Weight** | **Light Drinking** | **Moderate Drinking** | **Heavy Sessions** |
|-----------------|-------------------|---------------------|-------------------|
| **Under 130 lbs** | 300mg | 400mg | 500mg |
| **130-180 lbs** | 400mg | 500mg | 600mg |
| **Over 180 lbs** | 500mg | 600mg | 600mg |
```

**Why This Converts:**
1. User can FIND THEMSELVES in the table (personalization)
2. They get a SPECIFIC number for their situation
3. No ambiguity = higher CTR

**Critical Detail:** This appears AFTER the direct answer but BEFORE deep dives. It's the utility layer.

### Comparison Post Example

**DHM vs Milk Thistle - "Specific Use Cases" (Lines 229-275):**

```markdown
### When to Choose DHM
**Best For:**
1. **Social Drinkers** - Hangover prevention, etc.
2. **Binge Drinking Protection** - Acute liver protection
3. **Alcohol-Related Anxiety** - GABA receptor protection
4. **Athletic Performance** - Faster recovery

### When to Choose Milk Thistle
**Best For:**
1. **Chronic Liver Conditions** - Fatty liver disease, etc.
2. **General Liver Health** - Preventive protection
3. **Medication Users** - Protection from drug-induced damage
4. **Environmental Toxin Exposure** - Chemical protection
```

**The Insight:** Reader sees themselves in one of these boxes and knows which supplement is for them.

### Why 4-5 Segments = Perfect Number

- **Too few (2-3):** Feels oversimplified
- **Just right (4-5):** Covers 95% of use cases
- **Too many (8+):** Analysis paralysis

### Monetization Angle

Personalization tables are **affiliate link multipliers:**
