# Requirements: issue-363-dcni-bugs

## Goal

Fix 4 unrelated technical bugs causing indexing problems on dhmguide.com. Pure bug fixes; no behavioral changes. Each is independent and reversible. All ship in a single atomic PR (parent issue #363).

## Glossary

- **DCNI**: "Discovered – currently not indexed" — Google Search Console status.
- **308**: HTTP permanent redirect (preserves method, signals consolidation to Google).
- **307**: HTTP temporary redirect (Google does not consolidate signals across 307).
- **RFC 3986**: URI syntax spec. `%` is only valid as a percent-encoding prefix followed by exactly two hex digits; `'` is sub-delim, parser-unsafe in mixed pipelines.
- **Slug-filename drift**: state where on-disk JSON filename differs from `slug` field inside the JSON. Breaks prerender + dynamic import.

## User Stories

### US-1: Crawler-fetchable sitemap URLs
**As a** search engine crawler
**I want** every sitemap URL to be a valid RFC 3986 URI
**So that** I can fetch it without parser error

### US-2: Signal consolidation via 308
**As a** search engine consolidating ranking signals
**I want** non-www → www to redirect with 308 (permanent)
**So that** I can attribute traffic and links to the canonical host

### US-3: Single canonical for dosage calculator
**As a** search engine
**I want** only ONE indexable dosage-calculator route
**So that** I have a clear canonical and don't split ranking signals

### US-4: URL-safe slugs going forward
**As a** future maintainer
**I want** post slugs to use only `[a-z0-9-]`
**So that** I don't reintroduce these bugs

## Edge Cases

- **EC-1: Mixed encoding of `%`.** The `%` slug appears as raw `%` in source filenames and references, and as `%25` in some URL contexts (e.g. browser address bar, `curl -sI` output). Search-and-replace must handle BOTH. The 308 redirect's `source` field MUST use `%25` (URL-encoded form) so Vercel matches incoming requests with the literal `%` correctly.
- **EC-2: `metadata/index.json` is runtime-loaded.** Per research learnings: `src/newblog/utils/postLoader.js` reads this at runtime. Failing to update it = blog listing breakage. Same applies to `metadata/index.backup.json` (drift here re-introduces the bug on any future restore).
- **EC-3: `validate-posts.js` prebuild gate.** If a rename leaves the JSON invalid (slug ≠ filename), prebuild script fails and `npm run build` exits non-zero. Validates after rename.
- **EC-4: `cluster-config.json` is generator input.** Bug 1's slug appears at line 170; orphan-link-injector + sitemap generator read this. Stale reference = new slug excluded from cluster operations.
- **EC-5: Vercel dashboard race.** Bug 3's `vercel.json` rule must coexist with the dashboard "Preferred Domain" feature being OFF. If both active, Vercel community confirms undefined behavior. The fix is documented but the manual step is OUT of code.
- **EC-6: Catch-all redirect ordering.** Bug 3's `(.*)` source could in principle match Bug 1/Bug 2 slug-specific sources. It does NOT in practice because Bug 3 has a `host: dhmguide.com` predicate while Bug 1/2 do not. As defensive convention, place specific redirects BEFORE catch-all.
- **EC-7: Bug 4 implicit imports.** Pre-delete grep `grep -rn 'DosageCalculatorRewrite' src/` returned only `App.jsx:22`. Confirmed safe to delete. If a hidden import emerges in build, restore.
