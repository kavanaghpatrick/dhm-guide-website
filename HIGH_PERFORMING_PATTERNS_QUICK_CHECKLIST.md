# HIGH-PERFORMING PATTERNS - QUICK CHECKLIST

## Pre-Writing Checklist (Do This FIRST)

### Research Phase
- [ ] Find 3-5 actual GSC queries related to topic
- [ ] List variations of those queries (how much, when to, can I, etc.)
- [ ] Research competitor top 10 for content structure
- [ ] Plan which queries will get FAQs

### Content Planning
- [ ] Choose content type: Guide / Comparison / Review
- [ ] Select appropriate template (3 provided in full analysis)
- [ ] Plan personalization segmentation (4-5 user types)
- [ ] Map internal linking destinations (hub role)

## PATTERN 2 CHECKLIST: Personalization Tables

### Table 1: Primary Segmentation
- [ ] Choose primary segment dimension:
  - [ ] By weight/size (guides)
  - [ ] By use case (comparisons)
  - [ ] By budget (reviews)
- [ ] Create 4-5 tiers covering 95% of users
- [ ] Add specific, actionable column values

### Table 2: Secondary Segmentation (Optional)
- [ ] Add secondary dimension if applicable:
  - [ ] By drinking intensity (light/moderate/heavy)
  - [ ] By user type (social/chronic/athlete)
  - [ ] By price point (budget/moderate/premium)

### Affiliate Integration (if applicable)
- [ ] Add "Best Product" column to primary table
- [ ] Link each product to affiliate page
- [ ] Ensure links are relevant to segment
- [ ] Test links to ensure 301 redirects work

## PATTERN 3 CHECKLIST: Comprehensive FAQs

### Research & Compilation
- [ ] Export 10-20 relevant GSC queries
- [ ] Convert each query to FAQ format
- [ ] Write 2-3 sentence answer for each
- [ ] Aim for 10-15 FAQs total

### FAQ Organization
- [ ] Group FAQs into 2-3 logical categories
- [ ] Use H3 headers for FAQ titles
- [ ] Keep answers scannable (bullets, bold, numbers)
- [ ] Target 50-150 words per FAQ answer

### Query Matching (Critical!)
- [ ] FAQ #1 matches top query from GSC
- [ ] FAQ #2-5 match variations of top query
- [ ] FAQ #6-10 match secondary related queries
- [ ] FAQ #11-15 cover edge cases/nuances

**Example from DHM Dosage Guide:**
- Query: "how much dhm to take" (9 clicks, 34% CTR)

### Schema Markup
- [ ] Plan FAQ schema implementation
- [ ] Ensure each Q&A pair is semantic
- [ ] Prepare schema JSON for developer
- [ ] Verify in structured data tester

## PATTERN 4 CHECKLIST: Strategic Internal Linking

### Identify Hub Role
- [ ] What is this page's primary role?
  - [ ] Main hub (dosage guide, category page)
  - [ ] Secondary hub (comparison post)
  - [ ] Spoke (individual review)
- [ ] Plan inbound links (who links TO this?)
- [ ] Plan outbound links (what does this link TO?)

### Outbound Linking (From This Page)
- [ ] Identify 8-15 relevant internal pages
- [ ] Segment links by context:
  - [ ] 2-3 links to related deep dives
  - [ ] 2-3 links to individual products/posts
  - [ ] 1-2 links to topic cluster hub
  - [ ] 1-2 links to related comparisons
  - [ ] Remaining: affiliate product links
- [ ] Ensure links are contextual (not random)

### Inbound Linking (TO This Page)
- [ ] Document where this page should be linked from
- [ ] Create linking plan for other pages:
  - [ ] Do product reviews link here?
  - [ ] Do related comparisons link here?
  - [ ] Does topic hub link here?
- [ ] Add 2-4 outbound links in each of those pages

### Anchor Text Strategy
- [ ] Use descriptive anchor text (not "click here")
- [ ] Vary anchors (don't repeat exact phrase)
- [ ] Target primary keyword in 1-2 anchors
- [ ] Use secondary keywords in others

## PATTERN 5 CHECKLIST: Word Count & Scannability

### Content Length (By Type)
- [ ] **Guide:** Target 5,000-8,500 words
  - But MUST have: 6+ tables, 13+ FAQs, emojis
- [ ] **Comparison:** Target 3,500-4,500 words
  - With: 4+ comparison tables, 10 sections
- [ ] **Review:** Target 2,500-3,500 words
  - With: 2-3 tables, 3-5 FAQs, verdict box

### Scannability Elements
- [ ] Use table of contents (H2 headers)
- [ ] Add 1 table per 500-750 words
- [ ] Break text with H2-H4 subheadings every 200-300 words
- [ ] Use bullet points for lists (max 5 bullets per list)
- [ ] Bold key statistics and numbers
- [ ] Use emojis as visual anchors (15-25 per post)

### Mobile-First Check
- [ ] All tables render correctly on mobile?
- [ ] Emojis displaying properly?
- [ ] Bullet lists readable on small screens?
- [ ] Paragraph length <100 words?
- [ ] No giant text blocks without breaks?

## QUICK WINS: Apply to Existing Content (15 min each)

### Fastest wins for existing pages:

1. **Add Direct Answer Section**
   - Find top GSC query for page
   - Write 3-4 quick FAQs with emojis
   - Insert after intro, before main content
   - Time: 15 minutes
   - Expected impact: +100-150% CTR

2. **Add Personalization Table**
   - Identify primary user segmentation
   - Create 4-5 row table with specific values
   - Insert after direct answers
   - Time: 15-20 minutes
   - Expected impact: +200-300% conversions

3. **Add Internal Linking**
   - Identify 5-8 relevant internal pages
   - Add 2-3 contextual links throughout content
   - Time: 10-15 minutes
   - Expected impact: +0.5-1 position improvement

4. **Add FAQ Section**
   - Extract 5 most relevant GSC queries
   - Write Q&A for each (2-3 sentences)
   - Insert as H3 section
   - Time: 20 minutes
   - Expected impact: Rank for 8-10 related queries

## FINAL QUALITY GATE (Before Publishing)

### Ask Yourself
- [ ] Can someone answer their search query in first 100 words?
- [ ] Can they find themselves in personalization table?
- [ ] Could this page rank in Google's "People Also Ask"?
- [ ] Does this page feel fast/scannable on mobile?
- [ ] Are there 2+ conversion/link opportunities?
